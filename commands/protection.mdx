# Protection Commands

Protection commands help server administrators and moderators maintain order and enforce rules. These commands include basic moderation actions like kick, ban, and mute, as well as message management tools.

## Moderation Commands

| Command | Description | Usage | Example |
| ------- | ----------- | ----- | ------- |
| `kick` | Kick a member from the server | `!kick @user [reason]` | `!kick @user Breaking rules` |
| `ban` | Ban a member from the server | `!ban @user [reason]` | `!ban @user Repeated rule violations` |
| `unban` | Unban a user from the server | `!unban <user_id>` | `!unban 123456789012345678` |
| `mute` | Mute a member in the server | `!mute @user [duration] [reason]` | `!mute @user 1h Spamming` |
| `unmute` | Unmute a previously muted member | `!unmute @user` | `!unmute @user` |

## Warning System

| Command | Description | Usage | Example |
| ------- | ----------- | ----- | ------- |
| `warn` | Warn a member for rule violations | `!warn @user [reason]` | `!warn @user Inappropriate behavior` |
| `warnings` | View warnings for a member | `!warnings [@user]` | `!warnings @user` |
| `clearwarns` | Clear warnings for a member | `!clearwarns @user [warning_id]` | `!clearwarns @user 3` |

## Channel Management

| Command | Description | Usage | Example |
| ------- | ----------- | ----- | ------- |
| `purge` | Delete multiple messages at once | `!purge [amount]` | `!purge 10` |
| `slowmode` | Set slowmode for a channel | `!slowmode [seconds]` | `!slowmode 5` |
| `lock` | Lock a channel to prevent messages | `!lock [channel] [reason]` | `!lock #general Raid in progress` |
| `unlock` | Unlock a previously locked channel | `!unlock [channel]` | `!unlock #general` |

## Permissions Required

Most protection commands require specific Discord permissions:
- `kick` requires Kick Members permission
- `ban` and `unban` require Ban Members permission
- `mute` and `unmute` require Manage Roles permission
- `purge` requires Manage Messages permission
- `lock` and `unlock` require Manage Channels permission

Server administrators automatically have access to all these commands.
