# social notifications

automatically announce new posts from your favorite creators.

## supported platforms

there are eight social notification commands, each with the same subcommands.

- **instagram** - dispatched whenever a new post is uploaded.
- **tiktok** - dispatched whenever a new video is uploaded.
- **youtube** - dispatched whenever a new video or short is uploaded.
- **twitter** - dispatched whenever a new tweet is posted by a user.
- **reddit** - dispatched whenever a new post is made in a subreddit.
- **soundcloud** - dispatched whenever an artist releases a song or album.
- **pinterest** - dispatched whenever a new pin is saved to a board.
- **twitch** - dispatched whenever a streamer goes live.

## commands

the following examples use the `twitter` command, but the same syntax applies to the other social notification commands.

<details>
<summary>creating a new feed</summary>

```
,twitter add <channel> <username>
```

for youtube, use a channel url instead of a username:

```
,youtube add <channel> <channel_url>
```

for reddit, specify whether you're following a subreddit or user:

```
,reddit follow subreddit <channel> <subreddit>
,reddit follow user <channel> <username>
```

</details>

<details>
<summary>removing a social feed</summary>

use the `remove` subcommand to no longer receive notifications for a specific user.

<div className="border border-border rounded-lg overflow-hidden my-4">
  <div className="flex items-center justify-between bg-muted px-4 py-2">
    <span className="text-xs text-muted-foreground">syntax</span>
    <span className="text-xs text-muted-foreground">example</span>
  </div>
  <div className="p-4 border-t border-border">
    <code>
      ,twitter remove <span className="text-accent">(channel)</span>{" "}
      <span className="text-primary">(username)</span>
    </code>
  </div>
</div>

for reddit:

```
,reddit unfollow subreddit <channel> <subreddit>
,reddit unfollow user <channel> <username>
```

</details>

<details>
<summary>customizing the message</summary>

use the `message` subcommand to customize the message content that is displayed when a social notification is dispatched.

<div className="border border-border rounded-lg overflow-hidden my-4">
  <div className="flex items-center justify-between bg-muted px-4 py-2">
    <span className="text-xs text-muted-foreground">syntax</span>
  </div>
  <div className="p-4 border-t border-border">
    <code>
      ,twitter message <span className="text-primary">(username)</span>{" "}
      <span className="text-accent">(message)</span> [--pingable on|off]
    </code>
  </div>
</div>

this command is usually used to ping a role or user. in order for the message to actually notify the role or user, you must include the `--pingable on` flag in the command.

available placeholders vary by platform:

- **twitter**: `{username}`, `{text}`, `{url}`, `{retweets}`, `{likes}`, `{replies}`
- **instagram**: `{username}`, `{caption}`, `{url}`
- **youtube**: `{channel}`, `{title}`, `{description}`, `{url}`
- **tiktok**: `{username}`, `{description}`, `{url}`
- **pinterest**: `{username}`, `{description}`, `{url}`
- **soundcloud**: `{username}`, `{title}`, `{description}`, `{url}`, `{duration}`
- **twitch**: `{username}`, `{title}`, `{game}`, `{url}`, `{viewers}`
- **reddit**: `{subreddit}`, `{username}`, `{title}`, `{content}`, `{url}`, `{score}`, `{comments}`
</details>

<details>
<summary>viewing all users being monitored</summary>

use the `list` subcommand to view all users being monitored.

<div className="border border-border rounded-lg overflow-hidden my-4">
  <div className="p-4">
    <code>,twitter list</code>
  </div>
</div>
</details>

## specific commands

<details>
<summary>twitter retweets</summary>

by default, enovuh will send notifications for all tweets from an account. you can customize this with the mode command:

```
,twitter mode <username> <mode>
```

available modes:

- `all` - all tweets, retweets, and replies
- `tweets` - only original tweets
- `retweets` - only retweets
- `replies` - only replies
- `media` - only tweets containing media
- `tweets+media` - original tweets and media tweets
</details>

<details>
<summary>pinterest embeds</summary>

control whether pinterest notifications include rich embeds:

```
,pinterest embeds <on/off>
```

this setting applies to all pinterest notifications in your server.

</details>

<details>
<summary>reddit message commands</summary>

reddit has separate message commands for subreddits and users:

```
,reddit message subreddit <subreddit> <message>
,reddit message user <username> <message>
```

to view these messages:

```
,reddit message view_subreddit <subreddit>
,reddit message view_user <username>
```

</details>

<details>
<summary>youtube channel urls</summary>

when adding a youtube channel, you can use any format of youtube url:

- channel urls: `youtube.com/c/ChannelName`
- user urls: `youtube.com/user/Username`
- custom urls: `youtube.com/@ChannelName`
</details>
